About the author:

I am a 30 something system administrator living in Finland.

History:

I originally created this tool as a personal project to learn web development. At first it was just an automated email system to help system administrators keep track of common enterprise application updates. After creating the API I noticed that this could actually be extended and help a lot of people automate repetitive work.

My vision is to create a community run and trusted API that everyone can use to automate third-party application patching!

API:

Ideally you can use The Early Warning System API to fully automate third-party patching in SCCM/Intune for free. You can write your own scripts to pull the data from the API.

Use The Early Warning System at your own risk. I take no responsiblity for any issues that it might cause. It is up to you to verify downloaded files and build proper security checks if you decide to use it.

This Project does not host Applications directly. It is just providing direct download links to the vendors own websites. Downloading the applications does not grant you a license for the software. Please ensure you are properly licensed for all software you package and distribute!

So how does it work again?:

In a nutshell: This web application is reading the HTML of software vendor websites. Once changes are detected and a new version of tracked software is released it updates the database, api and emails all users about the new software.

Known issues:

- Sometimes the HTML in the software vendor website changes in a way that the webcrawler stops working. If this happens an automated email is sent to me and I will fix the issue, but this can understandably cause "outages".

- Unfortunately not all software vendors provide checksum data to verify downloaded files. This can potentially cause security issues. You should always verify the integrity of downloaded files.

How often does the system check for version changes?

The system checks vendor websites once every 24 hours.
